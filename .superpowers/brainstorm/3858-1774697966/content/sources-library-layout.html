来源库想做成哪一种“书库感”？

我把现在过于像筛选列表的问题拆成 3 个方向。它们都比现状更像产品，但气质完全不同。

A · 藏书墙 / Gallery Shelf

像书墙一样浏览

强调封面感和收藏感，第一眼更像个人书房，但会弱化筛选与来源元信息。

B · 阅览室 / Study Shelf

像安静的阅览室书架

保留书感与浏览节奏，也保留必要摘要和来源信息。我推荐这个，它最稳，不会做成样子货。

C · 索引书目 / Catalog Cabinet

像图书目录卡片柜

信息效率最高，最适合重度检索，但书感偏弱，容易又滑回“高级列表页”。

我建议先选 B

  • 最容易把来源库和 Reading 区分开
  • 比现在更像书库，但不会丢掉可读摘要
  • 可以在不大改数据结构的前提下落地

现状的问题

  • 左侧筛选太像后台侧栏
  • 右侧列表是一维信息流，不像藏书浏览
  • 来源库和 Reading 的气质差异还不够大
